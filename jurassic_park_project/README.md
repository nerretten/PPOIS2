# Лабораторная работа 1: Симуляция "Модель Юрского Периода"

Консольное приложение (CLI), реализующее объектно-ориентированную модель процесса палеонтологических раскопок, лабораторных исследований и управления музеем динозавров.

Проект разработан в рамках лабораторной работы по изучению принципов ООП (Объектно-Ориентированного Программирования), паттернов проектирования и модульного тестирования на языке Python.

## Описание предметной области

Пользователь берет на себя роль Главного Научного Сотрудника. Основная цель — исследовать Юрский период, выкапывая ископаемые, изучая их в лаборатории и создавая музейные экспонаты для получения финансирования.

**Основные механики:**

* **Полевые работы:** Раскопки в различных геологических формациях (Моррисон, Зольнхофен, Тяоцзишань). Успех зависит от уровня навыка исследователя и сложности породы. Раскопки тратят Энергию.
* **Лаборатория:** Очистка и анализ находок. Качественный анализ повышает общий уровень знаний человечества о Юрском периоде.
* **Музей:** Сборка полноценных скелетов из изученных костей. Качество (и ценность) экспоната зависит от степени сохранности костей и текущего уровня знаний о периоде.
* **Экономика:** Проведение выставок приносит бюджет и репутацию, позволяя продолжать исследования.
* **Система сохранений:** Весь прогресс (состояние объектов) автоматически сохраняется в формат JSON и восстанавливается при следующем запуске.

## Архитектура проекта

Код разделен на независимые слои, следуя принципу единой ответственности (Single Responsibility):

* `src/models.py` — **Доменная модель** Содержит классы сущностей (`Researcher`, `GeologicalFormation`, `Fossil`, `Museum`). Реализует инкапсуляцию — объекты сами управляют своим внутренним состоянием.
* `src/services.py` — **Бизнес-логика** Чистые классы-сервисы (`ExcavationService`, `MuseumService`), которые описывают правила взаимодействия между моделями. Здесь же реализован фасад `SimulationCore` и система сохранения `PersistenceService`.
* `src/main.py` — **Пользовательский интерфейс (CLI).** Контроллер, отвечающий за отрисовку меню, обработку ввода пользователя и перехват исключений.
* `tests/` — **Автотесты.** Набор юнит-тестов, проверяющих бизнес-логику и интерфейс с использованием мокирования.

## Установка и запуск

**Требования:**

* Python 3.10+
* Менеджер зависимостей Poetry

### Способ с использованием Poetry (Рекомендуемый)

1. Клонируйте репозиторий:
```bash
git clone https://github.com/nerretten/PPOIS2
cd jurassic_park_project

```


2. Установите зависимости:
```bash
poetry install

```


3. Запустите симуляцию:
```bash
poetry run python -m src.main

```

## Тестирование

Проект покрыт unit-тестами с использованием фреймворка `pytest`. Использовано мокирование (`unittest.mock.patch`) для имитации пользовательского ввода, изоляции файловой системы и генератора случайных чисел.

Для запуска тестов и получения отчета о покрытии выполните команду:

```bash
poetry run pytest --cov=src tests/  

```
